# マンスリー会員（継続寄付）— Pay.jp 実装キット

えひめ西条つながり基金の「マンスリー会員（毎月の継続寄付）」を、静的HP＋Cloudflare Workers＋Pay.jp で動かすための最小キットです。**個人用・法人用の2フォーム**を用意しています。まずは**テスト環境で通し**、動作を確認してから本番へ切り替えます。

## 構成

```
pay-jp-monthly/
├─ donate.html            … 個人用フォーム（HPに貼り付け）
├─ donate-corporate.html  … 法人用フォーム（HPに貼り付け）
├─ worker/
│  ├─ index.js            … 継続課金を作るCloudflare Worker（個人・法人 共通）
│  └─ wrangler.toml       … デプロイ設定
└─ README.md              … この手順書
```

処理の流れ：
`donate.html` / `donate-corporate.html`（payjp.jsでカードをトークン化）→ `Worker`（顧客＋月額プラン＋継続課金を作成）→ Pay.jp

---

## フォームの項目（コングラントの項目設定を踏まえて設計）

### 個人用（donate.html）

| 項目 | 必須/任意 |
|---|---|
| 毎月の金額（1,000/3,000/5,000円＋任意額） | 必須 |
| 寄付の方法（お名前を登録 ／ 匿名で寄付） | 必須（下記参照） |
| お名前・フリガナ・電話番号・ご住所 | 必須（**匿名を選んだ場合は非表示・不要**） |
| 性別・生年月日 | 任意 |
| 郵送物の送付先（住所と異なる場合） | 任意 |
| 広報物・お礼状等への氏名掲載 | 任意（既定：希望しない） |
| メールアドレス | 必須（匿名でも、確認・カード管理連絡のため必要） |
| 当基金を知ったきっかけ | 任意 |

**匿名寄付**：「匿名で寄付する」を選ぶと氏名・フリガナ・電話番号・住所の入力欄が消え、メールアドレスのみで申込めます。寄付金控除の**領収書は発行されません**（発行には氏名・住所が法的に必要なため）。控除が不要な方向けの選択肢です。

「応援したいテーマ」（プロジェクト選択）は、マンスリー会員が財団運営全体への支援であるため設けていません。

### 法人用（donate-corporate.html）

| 項目 | 必須/任意 |
|---|---|
| 毎月の金額（3,000/10,000/30,000円＋任意額） | 必須 |
| 法人名・フリガナ、ご担当者名・フリガナ | 必須 |
| ご担当者 部署・役職 | 任意 |
| メールアドレス・電話番号・所在地 | 必須 |
| 郵送物の送付先（所在地と異なる場合） | 任意 |
| 広報物・お礼状等への法人名掲載 | 任意（既定：希望しない） |
| 当基金を知ったきっかけ | 任意 |

法人用に匿名寄付の選択肢は設けていません（必要になれば個人用と同様に追加できます）。

## 手順

### 1. Pay.jp 側の準備
1. Pay.jp にログインし、**Visa/Mastercard が 1.5%（NPO料率）** になっているか確認。
2. **APIキー**を控える。まずは**テスト用**：
   - 公開鍵 `pk_test_…`（フロントで使用）
   - 秘密鍵 `sk_test_…`（Workerで使用・絶対に公開しない）
3. プランは**自動生成**されるので事前作成は不要（金額ごとに `monthly_<金額>` が作られます）。

### 2. Worker をデプロイ（Cloudflare・無料枠）
Node.js が入っている前提です。`worker/` フォルダで実行します。

```bash
# 初回のみ：Cloudflareにログイン
npx wrangler login

# シークレット鍵を登録（プロンプトに sk_test_… を貼る）
npx wrangler secret put PAYJP_SECRET_KEY

# wrangler.toml の ALLOW_ORIGIN を、フォームを置くサイトのURLに変更してから：
npx wrangler deploy
```

デプロイ後に表示される URL（例 `https://saijo-monthly-donation.xxxx.workers.dev`）を控えます。**個人用・法人用の両フォームで同じWorker URLを使えます。**

寄付者情報（氏名・住所など、寄付金控除の証明書や名簿のもとになるデータ）を残すため、**KV Namespace の作成を推奨**します：

```bash
npx wrangler kv namespace create DONORS
# 表示された id を wrangler.toml の [[kv_namespaces]] に貼り付けてからデプロイ
```

`wrangler.toml` 内の該当箇所のコメントを外し、id を設定してください。

> ターミナルを使いたくない場合：Cloudflareダッシュボード → Workers & Pages → Create → 「Hello World」Workerを作成 → コードを `index.js` の内容に置き換え → Settings で変数 `ALLOW_ORIGIN` と、暗号化変数 `PAYJP_SECRET_KEY` を登録、でも同じことができます（KVもダッシュボードから作成・バインド可能）。

### 3. フォームを設定
`donate.html` と `donate-corporate.html` それぞれの先頭 `CONFIG` を書き換えます（同じ値でOK）。

```js
var PUBLIC_KEY = "pk_test_…";                         // Pay.jp 公開鍵（テスト）
var WORKER_URL = "https://…workers.dev";              // 手順2で控えたURL
```

### 4. テスト
各HTMLをブラウザで開き、**テストカード**で申し込み：

- カード番号：`4242 4242 4242 4242`
- 有効期限：未来の日付（例 12/34）／ CVC：任意3桁（例 123）

個人用は「お名前を登録」「匿名」の**両方のパターン**を試してください。成功したら Pay.jp ダッシュボード（テスト）に **顧客・定期課金** が作られているか、KVを設定していれば寄付者記録も確認します。

### 5. 本番公開
1. `PUBLIC_KEY` を `pk_live_…` に、Workerの秘密鍵を `sk_live_…` に差し替え
   （`npx wrangler secret put PAYJP_SECRET_KEY` を本番鍵で再実行）。
2. `wrangler.toml` の `ALLOW_ORIGIN` を**本番HPのドメイン**にして再デプロイ。
3. `donate.html`（個人用）／`donate-corporate.html`（法人用）を既存の静的サイトの該当ページに貼り付け。

## 金額プランについて
- 個人：1,000／3,000／5,000円＋任意額。法人：3,000／10,000／30,000円＋任意額。
- 入力金額ごとに `monthly_<金額>` というプランを自動生成し、個人・法人で共有・使い回します。
- 下限1,000円・上限1,000,000円（`worker/index.js` で変更可）。

## まだ入っていない（次のフェーズ）
- サンクスメール／領収書の自動送信（決済成功をトリガに送る。住所等はKVに保存済みなので土台はできている）
- 解約・金額変更のフォーム
- 決済失敗の通知（Webhook）
- 寄付者データベースの本格的な管理画面（今はKVに個別レコードとして記録）

## 注意
- 秘密鍵（`sk_…`）は**フロントやGitに絶対に置かない**。Workerのsecretのみ。
- `ALLOW_ORIGIN` を本番ドメインに絞ると、他サイトからの不正POSTを防げます。
- 匿名寄付は氏名・住所を記録しないため、**領収書発行や名簿掲載の対象外**になります。運用ルールとして案内文に明記してください。
- これはテスト先行の最小構成です。本番公開前に必ずテスト環境で一連の動作を確認してください。
